Entities:

Some characters are reserved in HTML. To display them, write &entityName.
It is also possible to use entity numbers, &#entityNumber, which are harder to remember, but are supported almost everywhere.
The entity name of < is lt, and > is gt.
Another common entity is &nbsp, non-breaking space, which ensures that there wil be no linebreak at that space.

Block-level element:

A block-level element starts on a new line and takes up all the width available.
Use the div-element as a container for other elements to make them block-level.

Inline elements:

Inline elements do not start a new line, and only uses the width necessary. You can use the span-element to make for instance text-elements inline.

Classes:

You can define an element's class using class="*The class*" and then set a style for every element in that class in your css-document.
In the css-file the class is referred to using .*class name*, or in the case of an inline class, span.*class name*.
Note that the class name is case sensitive. It is also possible to have a single element be a part of several classes.

IDs:

The id attribute has to be unique within the HTML-file, and can be referred to in the CSS-file using #*The ID*. The id is case sensitive and cannot contain any white space.
Note the difference between class and id: An id only refers to a single element, wheras a class may refer to multiple elements.

Responsive Web Design:

Setting the Viewport:

The viewport sets the page's scaling and dimensions. Use the meta I have used in this document.

Responsive Images:

Often nice to use max-width:100%, which will make the image scale down if it has to, but never exceed its original size. The height can be set to auto.
By using the picture- and source-elements, it is possible to show different images for different browser window sizes.

Responsive Text:

By using the vw-unit, the text size will follow the size of the browser window.

Media Queries:

Can change the layout based on the size of the browser window. For instance make elements display horizontally on large screens and vertically on smaller screens.
First define how it should look on large screens, and then use
@media screen and (max-width: *number*px)
to define how it should look when the browser window is less than *number* pixels wide.

Semantic Elements:

A semantic element clearly describes its meaning to both the browser and the developer, unlike for instance div and span.

Section
- defines a section in the document.
Article
- specifies independent, self-contained content.
- should make sense on its own. Can be read independently of the rest of the webpage.
Header
- specifies a header for a document or section.
- should be used as a container for introductory content.
Footer
- specifies a footer for a document or section.
- should contain information about its containing element.
- typically contains the author of the document, copyright information, links to terms of use, contact information, etc.
Nav
- defines a set of navigation links.
- not all links should be in nav, only the major ones.
Aside
- defines some content aside from the content it is placed in (like a sidebar).
- should be related to the surrounding content.
Figure
- should group together image and figcaption.

There are also several other semantic elements, like time.

CSS

We will now have a closer look at some CSS-properties that we have not already covered.

Transparency:

When using rgba or hsla for color, you can add a fourth number which defines how transperent the color should be. 1 is normal and 0 is completely transperent.
If you are using a name for a color instead of an rgb/hex/hsl, then you can use the opacity-element to specify the transparency of the color.
NB! If you use the opacity property on a background element, the text within that element will inherit the same transparency making it hard to read. Using rgba or hsla prevents this inheritance.

Background-image:

Just like you can set a background-color, you can also set a Background-image. By default the image is repeated so it covers the entire element.
To only repeat an image horizontally, use background-repeat: repeat-x. To only repeat vertically, use background-repeat: repeat-y.
If you don't want the image to repeat, use background-repeat: no-repeat.
By using background-position: *location* you can place the image where you want.
Use background-attachment: fixed to fixate the background-image such that it stays the same place on the screen even when scrolling. Use background-attachment: scroll to make the image scroll like the rest of the page.

Borders:

First, you have to decide the style of the border, by using the border-style property. There are a number of different opportunities, including dotted, dashed, solid, double, groove, ridge, etc. It is also possible to use a mix. Check out this link to see all the different styles.
Border width can be chosen using the border-witdh property, and it is also possible to choose different widths for the different parts of the border. Same with the border color.
Instead of writing a line with different styles, colors, and/or widths in order to get the mixed border you want, you can write explicitly how each side should look by using border-top-style for instance.
For rounded corners on your border, use border-radius: *number*px.
